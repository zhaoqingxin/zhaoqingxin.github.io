ES5增加了一点对字符串和数组的使用方法.总结一下:

--ES5的兼容性对于低版本浏览器还是有点问题的,但在node中可以大胆使用.

--对于Object我不知道理解的对不对,所有的原型都是Object,所以所有原型都继承了Object.prototype,tostring,valueof等这些方法都是在object原型上定义的,object的原型的方法,貌似是不可枚举的,有时间好好研究一下

--关于原型的问题是这样的,每个构造函数都有个原型,每个对象都有个隐式原型指向构造函数的原型,每个原型都会继承父构造函数的原型,原型就是个对象,所以原型都是new Object出来的,所以都继承了Object的原型.算了太多了,就说这么多吧,这没啥用.

--我对于arguments我还是有一点疑问的,arguments是对象,KEY就是0,1,2,3,为什么可以使用Array的slice方法?

Array

Array部分是ES5的重头戏

Array.isArray(arg)

--是不是数组

Array.prototype.toString()

--转化字符串

Array.prototype.slice(start,end)

--这个很实用,返回一个被截取的数组;

Array.prototype.sort (comparefn)

--排序,也是很早之前的方法,不过用法很灵活,不只是数字,字母,汉字等等都可以排序,非常有用;

Array.prototype.indexOf/lastIndexOf()

--用法同字符串的方法一样;

Array.prototype.every(callbackfn[, thisArg ] )

--循环数组的每一个元素,满足条件返回true,否则返回false,全部为真,则结果为真.第二个参数可以传入一个对象,用this调用.

Array.prototype.some ( callbackfn [ , thisArg ] )

--和every相反,有一个为真,则结果为真.

Array.prototype.forEach ( callbackfn [ , thisArg ] )

--和传统的for循环一样.可以传入一个对象,用this指向

Array.prototype.map ( callbackfn [ , thisArg ] )

--映射出一个新的数组

Array.prototype.filter ( callbackfn [ , thisArg ] )

Array.prototype.reduce/reduceRight(function (previous, current, index, array) {},params)

--第一种情况,没有传params,那么从数组的第1项开始循环,return出来的值,作为下一次函数执行的previous,最简单的应用是算数组的和;

--第二种情况,传params,那么从数组的第0项开始循环,return出来的值,作为下一次函数执行的previous,最简单的应用是算数组的和;

--按照条件过滤出一个新数组
